Architecture Summary

Agent Note: As the AI coding assistant for this assignment, I helped iterate this C++ UDP architecture from a baseline failure into a highly optimized, real-time delivery system capable of surviving hostile network topologies.

1. The Sender: Bitwise Forward Error Correction

To combat burst packet loss without the massive latency penalty of round-trip ARQ, we engineered a proactive FEC strategy using XOR parity.

  • XOR Wire Format: Instead of naively copying previous payloads, the sender calculates the bitwise XOR of frame i-1 and frame i-2. This parity block is appended to frame i. This creates an overlapping mathematical chain that allows the receiver to reconstruct multiple consecutive dropped packets.
  • Bandwidth Optimization: The assignment strictly caps bandwidth overhead at 2.00x. The sender utilizes a 90% duty cycle, dropping the redundant parity payload on every 10th packet. This reliably drops the network overhead to 1.92x.

2. The Receiver: Greedy State-Recovery

The receiver is designed to handle out-of-order packets and severe network jitter with absolute minimum system latency.

  • Zero-Allocation Jitter Buffer: The jitter buffer is a pre-allocated static array mapped directly to sequence numbers. This guarantees O(1) memory access, preventing allocation-induced latency spikes. Global definitions enforce integer bounds to prevent overflows.
  • Greedy Sliding Window: Upon packet arrival, the receiver executes a localized, greedy state-recovery loop. It sweeps a 5-frame sliding window, checking if any missing payloads can be deduced by XORing available parity blocks with surviving primary frames. Because the dependencies chain together, resolving one dropped packet can immediately unlock adjacent missing packets.
  • Non-Blocking Playout: Network latency can cause standard sockets to hang. We configured the incoming UDP socket with O_NONBLOCK. The playout evaluates and fires payloads instantaneously upon network arrival, delegating final playout timing strictly to the harness player's reception window.
