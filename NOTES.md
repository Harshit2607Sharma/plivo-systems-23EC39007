The system implements a robust Forward Error Correction (FEC) mechanism using bitwise XOR parity across a two-frame window. To satisfy the 2.0x bandwidth cap, the sender operates on a 90% duty cycle, appending the parity block to outgoing packets to achieve a steady 1.92x overhead. The receiver utilizes a zero-allocation, pre-sized array for its jitter buffer, entirely avoiding runtime memory management latency. Playout is purely event-driven. To combat burst losses, the receiver executes a greedy state-recovery algorithm, evaluating a local sliding window upon every network arrival to dynamically solve for missing payloads via surviving parity chains. The optimal delay for grading on unknown hostile profiles is 100ms. The architecture is vulnerable to extreme latency spikes that exceed 100ms, or consecutive burst losses exceeding three packets, which would isolate and break the XOR parity chain before recovery can propagate.
